# 🧠 Контекст проекта Plaud2Obsidian

## Что это и зачем

Персональная база знаний по паттерну Андрея Карпаты (llm-wiki). Источник данных — устройство **Plaud** (диктофон/рекордер встреч). Цель — полностью автоматизировать путь: **голос → транскрипт → атомарные заметки → Obsidian**.

---

## Архитектура (3 слоя по Карпаты)

```
Plaud2Obsidian/
├── AGENTS.md          ← конституция агента (этот файл)
├── index.md           ← каталог всех wiki-страниц
├── log.md             ← хронологический журнал действий
├── .mcp.json          ← подключение DeepWiki MCP сервера
├── raw/               ← READONLY: сырые транскрипты из Plaud
└── wiki/
    ├── people/        ← профили людей
    ├── projects/      ← проекты
    ← ideas/         ← атомарные идеи
    ├── meetings/      ← встречи
    └── synthesis/     ← аналитика и ответы на запросы
```

**Правило:** `raw/` = только читать. `wiki/` = агент пишет.

---

## Пайплайн (как это работает сейчас)

### Шаг 1 — Получение транскрипта из Plaud
Через **неофициальный API** (Bearer-токен из localStorage браузера):
- `GET {PLAUD_API_DOMAIN}/file/simple/web` — список записей
- `GET /file/detail/{file_id}` — транскрипт + AI-саммари

Токен хранится в `.env`: `PLAUD_TOKEN=bearer eyJ...`, `PLAUD_API_DOMAIN=https://api-euc1.plaud.ai`

Скрипт выгрузки: `scripts/fetch_plaud.py` (сохраняет `.md` файлы в `raw/`)

### Шаг 2 — Атомизация (обработка)
Агент читает файл из `raw/`, определяет контекст и создаёт заметки в `wiki/` через **ObsidianDataWeave** (github.com/howdeploy/ObsidianDataWeave) — инструмент для нарезки длинного текста на атомарные заметки по методологии MOC + Zettelkasten.

Логика обработки описана в `AGENTS.md` (режим Ingest).

### Шаг 3 — Доставка в Obsidian
Git: сервер делает `git commit && git push` → Obsidian на телефоне/ПК подтягивает через плагин Obsidian Git.

## Целевая инфраструктура (сервер)

- **VPS:** Хельсинки, 1 ядро, 2 ГБ RAM, 30 ГБ NVMe (369 ₽/мес)
- **Агент:** OpenClaw — автономный ИИ-агент, общение через Telegram-бот
- **LLM:** внешний API (OpenAI / Gemini / OpenRouter) — не локально, т.к. RAM не позволяет
- **MCP:** `https://mcp.deepwiki.com/mcp` — для анализа GitHub-репозиториев прямо из агента

Пока сервер освобождается (переезжает старый проект).

## Инструменты и их роли

| Инструмент | Роль |
|------------|------|
| **Plaud API** (Bearer) | Выгрузка транскриптов |
| **ObsidianDataWeave** | Атомизация текста в Obsidian-заметки |

## Как работают режимы агента

**Ingest** — `"Обработай новую запись"` → агент читает `raw/`, создаёт заметки в `wiki/`, обновляет `index.md` и `log.md`.

**Query** — `"Что обсуждалось с Алексом?"` → агент читает `index.md` → нужные страницы `wiki/` → даёт ответ со ссылками. Хороший ответ предлагает сохранить в `wiki/synthesis/`.

**Lint** — `"Проведи уборку"` → агент ищет сирот, противоречия, пробелы → пишет отчёт в `log.md`.
